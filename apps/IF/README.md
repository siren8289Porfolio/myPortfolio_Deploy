# 고령자 노동·돌봄 규제 AI 판단 시스템 PRD

---

## 문제 정의

현행 고령자 노동 및 돌봄 관련 규제는 연령과 사업 유형 중심으로 설계되어 있으며,

개인의 **건강 상태·근로 가능 수준·직무 위험도**를 종합적으로 고려할 수 있는 정량적 판단 수단이 부재하다.

이로 인해 행정 현장에서는

- 담당자의 경험과 감각에 의존한 판단이 반복되고
- 동일 조건 신청자 간 판단 일관성이 낮으며
- 사고 발생 시 책임 소재가 불명확해지는 문제가 지속된다.

특히 현재 제도는 **직무 배치 이전에 위험도를 수치로 확인할 방법이 없고**,

산업재해 통계 또한 사후 집계 중심으로 활용되고 있어 **사전 예방적 규제 작동이 불가능한 구조**이다.

---

## 목표 (Goals)

- 고령자 노동 판단을 **정성적 판단 → 수치 기반 판단**으로 전환
- 개인별 위험도를 **퍼센트(%) 단위로 즉시 산출**
- 담당자가 법령 범위 내에서 **빠르게 판단할 수 있는 기준 제공**
- 판단 근거를 기록하여 사후 책임·정책 평가 가능 구조 확보
- **직무 추천 없이** 판단 보조 역할에 집중

---

## 타깃 사용자 (Persona)

### 1. 노인일자리 운영 담당자

- 고령자 배치 가능 여부를 결정해야 하는 공공 실무자
- 니즈:
    - “이 사람을 써도 되는지”를 **숫자로 확인**
    - 사고 발생 시 판단 근거로 남길 수 있는 기록

### 2. 정책 담당 공무원

- 규제혁신 정책 성과를 검증하는 담당자
- 니즈:
    - AI 판단 결과를 퍼센트로 비교
    - 정책 도입 전·후 효과 분석

※ **직무 추천·매칭 담당자는 핵심 타깃에서 제외**

---

## User Story

```
As a 노인일자리 운영 담당자
I want to 신청자의 상태를 입력하면
해당 직무에 대해 위험도가 몇 퍼센트인지 즉시 확인하고 싶다
So that 직무 배치 여부를 법령 범위 내에서 빠르게 판단할 수 있다

```

---

## 목표 (Non-Goals)

- ❌ 직무 추천, 우선순위 제안, 매칭 기능 제공하지 않음
- ❌ AI가 허용/불허를 자동 결정하지 않음
- ❌ 의료적 진단 또는 치료 판단 수행하지 않음
- ❌ 민간 HR 채용·평가 시스템으로 확장하지 않음
- ❌ 복수 선택지 비교 UX 제공하지 않음

---

## 기능 요구사항 (Functional Requirements)

### 1. 신청자 상태 입력

- 연령
- 신청 직무
- 신체 기능 수준
- 주요 질환 여부
- 근무 가능 시간/강도

---

### 2. 위험도 퍼센트 산출 (핵심 기능)

- 개인 상태 + 직무 위험도를 종합하여
    - **총 위험도 %** 산출
- 결과는 단일 수치 + 보조 지표로 제공
    - 예:
        - 총 위험도: **38%**
        - 주요 기여 요인: 체력 / 반복 작업 / 사고 이력

---

### 3. 판단 구간 표시

- 퍼센트 구간에 따른 해석 가이드 제공
    - 0~40% : 상대적으로 낮은 위험
    - 41~60% : 주의 필요
    - 61% 이상 : 고위험

※ **결정은 하지 않고, 해석만 제공**

---

### 4. 판단 기록 저장

- 입력값
- 산출된 퍼센트
- 담당자 확인 여부
- 타임스탬프

---

## 기술 요구사항 (Technical Requirements)

- 공공 통계 데이터 기반 산출 구조
- 설명 가능한 점수 산출 로직
- 규칙 기반 + 통계 기반 혼합 모델
- 결과 산출 시간 3초 이내
- 모든 결과 로그 변경 불가 저장
- 개인정보 비식별 처리

---

## Edge Case & Risks

### Edge Case

- 일부 건강 정보 미입력
- 고령 + 고위험 직무 조합
- 동일 조건 신청자 반복 입력
- 담당자 판단과 수치 간 괴리 발생

### Risks

- 퍼센트 결과의 과신
- 점수 해석 오류
- 차별 논란 발생 가능성

→ 대응:

- “판단 참고용 지표” 명시
- 퍼센트 산출 근거 항목 노출
- 최종 판단 주체는 인간으로 고정

---

## Acceptance Criteria

- 신청자 1인 위험도 산출 ≤ **3분**
- 퍼센트 결과 응답 ≤ **3초**
- 담당자 판단 기준으로 사용 가능
- 모든 판단 기록 추적 가능
- 법 개정 없이 현행 제도 내 사용 가능
