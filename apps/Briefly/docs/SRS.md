## 2. 시스템 개요

**Briefly**는 일반 사용자가 투자상품을 탐색하고, 관심상품 등록과 모의가입 신청을 할 수 있으며, 운용 브리프와 위험 알림을 확인할 수 있는 Servlet/JSP 기반 B2C 투자상품 관리 시스템이다.

관리자는 상품, 운용 브리프, 위험 알림, 모의가입 신청 상태를 관리한다.

## 7. 비기능 요구사항

| ID | 구분 | 요구사항 |
| --- | --- | --- |
| NFR-001 | 인증 | 시스템은 Session 기반 인증을 사용해야 한다. |
| NFR-002 | 권한 | 시스템은 관리자 기능에 ADMIN 권한 사용자만 접근하게 해야 한다. |
| NFR-003 | 보안 | 시스템은 비밀번호를 평문으로 저장하지 않아야 한다. |
| NFR-004 | 구조 | 시스템은 Servlet → Service → DAO → DB 구조로 분리되어야 한다. |
| NFR-005 | 사용성 | 사용자는 상품 상세 화면에서 관심 등록과 모의가입 신청을 3단계 이내로 완료할 수 있어야 한다. |
| NFR-006 | 예외처리 | 시스템은 입력 오류 발생 시 JSP 화면에 오류 메시지를 표시해야 한다. |

## 12. 비즈니스 규칙

| ID | 규칙 |
| --- | --- |
| BR-001 | 비로그인 사용자는 관심상품과 모의가입 기능을 사용할 수 없다. |
| BR-002 | 동일 사용자는 동일 상품을 관심상품으로 중복 등록할 수 없다. |
| BR-003 | 모의가입 신청 기본 상태는 PENDING이다. |
| BR-004 | 신청 상태는 PENDING, APPROVED, REJECTED, CANCELED 중 하나여야 한다. |
| BR-005 | 사용자 화면에는 ACTIVE 상태 상품만 노출한다. |
| BR-006 | 관리자 기능은 ADMIN 권한 사용자만 접근할 수 있다. |
| BR-007 | 위험등급은 1~5 사이 값만 허용한다. |
| BR-008 | 상품은 삭제하지 않고 INACTIVE 상태로 비활성화한다. |
