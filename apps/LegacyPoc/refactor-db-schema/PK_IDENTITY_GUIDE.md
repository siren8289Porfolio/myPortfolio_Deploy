# PK / 자동 증가 ID 설계 가이드

`warehouse_io_id`, `investor_id` 등 식별자 컬럼 설계. DB별 공식 문서와 이 프로젝트 적용 방식을 정리한다.

## 공식 문서

| DB | 문서 | 핵심 |
|----|------|------|
| **MariaDB** | [AUTO_INCREMENT](https://mariadb.com/kb/en/auto_increment/) | 정수 타입 + **KEY**(PK/INDEX) 필수. 명시적 값 INSERT 가능 |
| **MySQL** | [AUTO_INCREMENT](https://dev.mysql.com/doc/refman/8.4/en/example-auto-increment.html) | 새 row에 고유 번호 자동 생성. `0`/`NULL`이면 다음 번호 할당 |
| **PostgreSQL** | [Identity Columns](https://www.postgresql.org/docs/current/sql-createtable.html#SQL-CREATETABLE-PARMS-GENERATED-IDENTITY) | `GENERATED ALWAYS AS IDENTITY` vs `GENERATED BY DEFAULT AS IDENTITY` |
| **Oracle** | [Identity Column](https://docs.oracle.com/en/database/oracle/oracle-database/19/sqlrf/CREATE-TABLE.html#GUID-8B939D54-8E0E-4C97-806B-50D0B9D8F5C4) 또는 [CREATE SEQUENCE](https://docs.oracle.com/en/database/oracle/oracle-database/19/sqlrf/CREATE-SEQUENCE.html) | Identity clause 또는 Sequence + trigger/기본값 |

## 이 프로젝트 ID 설계

| 테이블 | PK 컬럼 | 시드 시작값 | 메모리 Repository 시퀀스 | 용도 |
|--------|---------|-------------|--------------------------|------|
| `tb_warehouse_io` | `warehouse_io_id` | 1001 | `AtomicLong(1000)` → 첫 ID **1001** | 입출고 |
| `tb_investor` | `investor_id` | 2001 | `AtomicLong(2000)` → 첫 ID **2001** | 투자자 |

두 도메인 ID 범위를 분리해 데모·로그에서 구분하기 쉽게 한다.

## DB별 구현 (본 프로젝트 선택)

### MariaDB / MySQL — `AUTO_INCREMENT`

```sql
warehouse_io_id BIGINT NOT NULL AUTO_INCREMENT,
CONSTRAINT pk_tb_warehouse_io PRIMARY KEY (warehouse_io_id)
) ENGINE=InnoDB AUTO_INCREMENT=1001;
```

| 규칙 | 설명 |
|------|------|
| 타입 | `BIGINT` (정수) |
| KEY | `PRIMARY KEY` 필수 — AUTO_INCREMENT 컬럼은 인덱스되어야 함 |
| 명시 INSERT | `INSERT ... (warehouse_io_id, ...) VALUES (1001, ...)` **가능** |
| 다음 번호 | 명시 INSERT 후 `MAX(id)+1`로 자동 갱신 |
| JPA | `@GeneratedValue(strategy = GenerationType.IDENTITY)` |

공식: MariaDB/MySQL은 PK 없이 AUTO_INCREMENT만 단독 사용 불가.

### PostgreSQL — `GENERATED BY DEFAULT AS IDENTITY`

```sql
warehouse_io_id BIGINT GENERATED BY DEFAULT AS IDENTITY,
CONSTRAINT pk_tb_warehouse_io PRIMARY KEY (warehouse_io_id)
```

| 모드 | 명시 INSERT | 용도 |
|------|-------------|------|
| `GENERATED **ALWAYS** AS IDENTITY` | 불가(override 시 특수 문법) | DB만 번호 생성 |
| `GENERATED **BY DEFAULT** AS IDENTITY` | **가능** | 시드 1001/2001 + JPA INSERT 병행 ← **본 프로젝트** |

시드 적용 후 시퀀스 동기화 (`data.sql`):

```sql
SELECT setval(pg_get_serial_sequence('tb_warehouse_io', 'warehouse_io_id'),
              GREATEST((SELECT COALESCE(MAX(warehouse_io_id), 1) FROM tb_warehouse_io), 1002));
```

다음 자동 INSERT는 **1003**부터 시작한다.

### Oracle — Identity Column (기본)

```sql
warehouse_io_id NUMBER(19) GENERATED BY DEFAULT ON NULL AS IDENTITY NOT NULL,
CONSTRAINT pk_tb_warehouse_io PRIMARY KEY (warehouse_io_id)
```

| 절 | 설명 |
|----|------|
| `BY DEFAULT ON NULL AS IDENTITY` | NULL이면 시퀀스 값, **명시값 INSERT 허용** (시드용) |
| `ALWAYS AS IDENTITY` | 명시 INSERT 제한 — 시드와 충돌하므로 미사용 |
| 타입 | `NUMBER(19)` — Java `Long` / JPA `Long`과 호환 |

### Oracle — SEQUENCE 방식 (대안, 참고용)

Identity 대신 전통적 Oracle 패턴. `refactor-db-schema/oracle/sequence-alternative.sql` 참고.

```sql
CREATE SEQUENCE seq_warehouse_io START WITH 1001 INCREMENT BY 1 NOCACHE NOCYCLE;
-- INSERT 시 seq_warehouse_io.NEXTVAL 또는 JPA @SequenceGenerator
```

| 비교 | Identity Column | SEQUENCE |
|------|-----------------|----------|
| DDL | 테이블에 포함 | 별도 객체 |
| JPA | `GenerationType.IDENTITY` | `GenerationType.SEQUENCE` + `@SequenceGenerator` |
| 이 프로젝트 | **채택** | 참고만 |

## ALWAYS vs BY DEFAULT — 왜 BY DEFAULT?

```
시드 data.sql:  INSERT ... warehouse_io_id = 1001  (명시)
JPA save:       INSERT ... warehouse_io_id 생략   (자동)
```

| DB | 선택 | 이유 |
|----|------|------|
| PostgreSQL | `BY DEFAULT AS IDENTITY` | 시드 + JPA 동시 지원 |
| Oracle | `BY DEFAULT ON NULL AS IDENTITY` | 동일 |
| MariaDB/MySQL | `AUTO_INCREMENT` | 기본적으로 명시 INSERT 허용 |

`ALWAYS` / `ALWAYS AS IDENTITY`만 쓰면 시드 SQL에서 ID를 넣기 어렵다.

## JPA 매핑 (`*JpaEntity`)

```java
@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)
@Column(name = "warehouse_io_id")
private Long warehouseIoId;
```

- MariaDB / MySQL / PostgreSQL / Oracle Identity → 모두 `GenerationType.IDENTITY`
- Oracle SEQUENCE 방식으로 바꿀 때만 `SEQUENCE` + `@SequenceGenerator` 필요

## INSERT 동작 요약

| 시나리오 | warehouse_io_id | 결과 |
|----------|-----------------|------|
| JPA `save()` (id=null) | 생략 | DB가 다음 번호 할당 |
| 시드 `data.sql` | 1001, 1002 | 명시값 그대로 저장 |
| 다음 JPA insert | 생략 | 1003 (시드·setval 이후) |

## 체크리스트

- [ ] PK 컬럼 `NOT NULL`
- [ ] 정수 타입 (`BIGINT` / `NUMBER(19)`)
- [ ] 시드 명시 INSERT가 필요하면 `BY DEFAULT` / `AUTO_INCREMENT` (not `ALWAYS`)
- [ ] PostgreSQL 시드 후 `setval`로 시퀀스 동기화
- [ ] MariaDB/MySQL 테이블 `AUTO_INCREMENT=1001` / `2001` (선택, 초기값 힌트)
