# DDL / 테이블 생성 가이드

리팩터·레거시 프로젝트의 DB 스키마 정의 요약. 각 DB 공식 `CREATE TABLE` 문법을 따른다.

## 공식 문서

| DB | CREATE TABLE 문서 | 주요 키워드 |
|----|-------------------|-------------|
| **MariaDB** | [CREATE TABLE](https://mariadb.com/kb/en/create-table/) | `IF NOT EXISTS`, `NOT NULL`, `DEFAULT`, `AUTO_INCREMENT`, `PRIMARY KEY`, `CHECK` |
| **MySQL** | [CREATE TABLE](https://dev.mysql.com/doc/refman/8.4/en/create-table.html) | `IF NOT EXISTS`, `NOT NULL`, `DEFAULT`, `AUTO_INCREMENT`, `PRIMARY KEY`, `CHECK` (8.0.16+) |
| **PostgreSQL** | [CREATE TABLE](https://www.postgresql.org/docs/current/sql-createtable.html) | `IF NOT EXISTS`, `NOT NULL`, `DEFAULT`, `PRIMARY KEY`, `GENERATED BY DEFAULT AS IDENTITY`, `CHECK` |
| **Oracle** | [CREATE TABLE](https://docs.oracle.com/en/database/oracle/oracle-database/19/sqlrf/CREATE-TABLE.html) | `NOT NULL`, `DEFAULT`, `PRIMARY KEY`, `GENERATED BY DEFAULT ON NULL AS IDENTITY`, `CHECK`, `VARCHAR2` |

## PK / Identity 방식 (DB별)

상세: [PK_IDENTITY_GUIDE.md](./PK_IDENTITY_GUIDE.md)

```sql
-- MariaDB / MySQL  (AUTO_INCREMENT 문서)
warehouse_io_id BIGINT NOT NULL AUTO_INCREMENT,
CONSTRAINT pk_tb_warehouse_io PRIMARY KEY (warehouse_io_id)
) AUTO_INCREMENT=1001;

-- PostgreSQL  (Identity Columns 문서)
warehouse_io_id BIGINT GENERATED BY DEFAULT AS IDENTITY,
CONSTRAINT pk_tb_warehouse_io PRIMARY KEY (warehouse_io_id)

-- Oracle  (identity clause)
warehouse_io_id NUMBER(19) GENERATED BY DEFAULT ON NULL AS IDENTITY NOT NULL,
CONSTRAINT pk_tb_warehouse_io PRIMARY KEY (warehouse_io_id)
```

`BY DEFAULT` / `AUTO_INCREMENT` — 시드 `1001`·`2001` 명시 INSERT + JPA 자동 ID 병행.

## 적용 순서

```bash
# 1. schema.sql (CREATE TABLE, PK, CHECK, INDEX)
# 2. data.sql (시드 INSERT — 멱등)
# 3. explain-verify.sql (인덱스 사용 여부 EXPLAIN 확인)
# 4. data.sql 재실행 (Upsert 멱등 검증 — row 수 불변)
```

각 모듈 `src/main/resources/schema.sql`은 `refactor-db-schema/<dialect>/schema.sql`과 동기화한다.
